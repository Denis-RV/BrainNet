### Вопросы по ООП:
- Java - это ООП язык. Как вы это понимаете?
```
ООП парадигма позволяет переложить объекты из нашего мира в наш код. Когда мы пишем программу, мы устанавливаем связи и взаимодействия между объектами подобно тому, как они взаимодействуют в реальном мире. 
```
- Преимущества и недостатки ООП?
```
Преимущества: Программы легко воспринимать, можно без труда понять чужой код, потому что все консепты имитирует то, что мы видим в реальной жизини. Код легко использовать повторно, благодаря наследованию. Также относительно легко работать над большими проектами.
Недостатки: Избыточность кода, особенно когда нам нужна маленькая программка. Это влияет и на память, и на скорость разработки и на понимание.
```
- Что такое инкапсуялция? 
```
- Инкапсуляция. 1)Позволяет закрывать наш объект для неправильного использования путем модификатора доступа. Если мы хотим добавить фото, обозначающее дату создания объекта, если мы не закроем это поле модификатором доступа, любой программист может использовать это поле как хочет и цель этого поля теряется. Таким образом мы защищаем наши объекты от неправильного доступа и оставляем их открытыми для расширения. 2)Это способность объектов скрывать часть своего состояния и поведения от других объектов, предоставляя внешнему миру только определённый интерфейс взаимодействия с собой.Например, вы можете инкапсулировать что-то внутри класса, сделав его приватным (private) и скрыв доступ к этому полю или методу для объектов других классов. Более открытый режим видимости protected сделает это полеьили метод доступным в подклассах
```
- Что такое наследование?
```
- Наследование. Один объект расширяет функциональность другово, добавляя собственные, уникальные характеристики. Допустим нам надо создать какую-нибудь гитару. Она может иметь разную форму, состоять из разного материала, по-разному стоить, но это должна быть гитара, а не труба, и при проектировании мы берем за гитару некую абстракцию (родительский класс), который имеет такие поля, как цена, материал, форма и имеет методы: сыграть такую-то ноту, настроить тональность.
```
- Что такое полиморфизм?
```
- Полиморфизм. Позволяет использовать одно имя для задания общих для классов действий. Например интерфейс "Издать звук" можно будет использовать как для класса "Птица", так и для класса "Слон".
```
- Что такое абстракция?
``` 
- Абстракция данных - 1)выделение значимых характеристик объектов. На том же примере с гитарой мы выделили значемые характеристики, которые мы объявили в абстрактном классе. Это переменные и назавания методов. 2)Это модель некоего объекта или явления реального мира, откидывающая незначительные детали, не играющие существенной роли в данном контексте.
```
- Что такое модификатор доступа? Какие они есть и для чего используются? 
```Модификаторы доступа нужны для защиты данных, с их помощью мы и используем инкапсуляцию. Модификаторов всего четыре:
1. public - класс доступен в любом метсе для всех
2. private - доступ внутри класса
3. package(default) - внутри пакетов.
4. protected - доступ внутри пакетов и доступ для наследников.

```

- Как устроена память в Java?
```
Существует Stack (ограничен 1 мб) и Heap(куча, ограничена размером оперативной памяти). Примитивные типы и ссылки на объекты хранятся в стеке. При создании объекта переменной присваевается адресс этого объекта, а сам объект помещается в кучу. Когда мы сравниваем два объекта чере знак сравнения, мы сравниваем их адреса в стеке, и значение true возвращается лишь в случае, когда две ссылки указывают на один объект. 
```
- Как сборщик мусора понимает, что нужно удалить объект? Если объекты ссылаются друг на друга, уничтожит ли их сборщик мусора?
```
Сборщик муссора удаляет объект, когда на него нет ссылок из стека.

Если объекты ссылаются друг на друга, но ни на один их этих объектов нет ссылок из стека, то они уничтожатся.
```

- Что такое string pool?
```
string-pool — структура в памяти, хранящая массив всех строк-литералов программы.  
String.intern(), соответственно, вернет строку из пула, при наличии таковой.

Пул строк - это набор строк, который хранится в памяти java heap. Если строка создается как литерал, то ищется поле соответсвия и из пула либо вернется ссылка, либо создаться строка и вернется ссылка. Если использовать конструктор стринга, то не будет искаться такой строки в пуле, но если вызвать у такой строчки intern(), то будет обращение к пулу. Пример:
String s1 = "Cat";
String s2 = "Cat";
String s3 = new String("Cat");
String s4 = s3.intern();
//Две ссылки указывают на одну строку в пуле:
System.out.println(s1==s2); //true
//s3 не имеет отношения к пулу
System.out.println(s1==s3); //false
//s4 ссылается на строку s3 в пуле
System.out.println(s1==s4); //true
```

##### Обработка исключений
- Что вы можете сказать об исключениях? Какие они бывают?
```
-   Checked exceptions (проверяемые исключения). В JDK представлены классом Exception. Исключения, которые нельзя проигнорировать, их обязательно нужно обрабатывать, либо специфицировать в сигнатуре метода, для обработки выше. Как правило, считаются дурным тоном, т.к код со мн-вом конструкций try..catch плохо читабелен, к тому же добавление новых пробрасываемых исключений в сигнатуре метода может сломать контракт вызова у пользователей данного метода.
-   Unchecked exceptions (непроверяемые исключения). В JDK это класс RuntimeException. Можно игнорировать, не требуют обработки через try..catch, или указания в сигнатуре через throws. Минус такого подхода — у вызывающей стороны нет никакого понимания, как обрабатывать ситуацию, когда под капотом «рванет»
-   Error — ошибки, кидаемые JVM в результате нехватки памяти (OutOfMemoryError), переполнения стэка (StackOverflowError) и.т.д
```

### Collection Framework:
- Что делают методы equals() и hashCode()? 
```
Метод equals по умолчанию делает то же, что и знак равенства: он сравнивает ссылки двух объектов. Чтобы этот метод сравнивал объекты по значению полей, необхоидоме этот метод переопределить.

Метод hashCode служит для целочисленного представления объекта. Для хеш-кода должно выполнятся правило: Если объекты равны по equals(), то и хеш-коды у них должны быть равны, но не наоборот.

Если объекты не равны по результатам выполнения метода equals, тогда их hashcode могут быть как одинаковыми, так и разными. Однако для повышения производительности лучше, чтобы разные обхъекты возвращали разные коды.

У двух разных объектов может совпасть хеш-код, т.к. этот хеш-код типа int и принимает не больше 2 млрд значений, а объектов может быть создан больше 2 млрд, это называется коллизией.

У одинакоыйх строк одинаковый хеш-код
```

```
ArrayList - это динамический массив, который увеличивается по мере добавления элементов. LinkedList - это связный список, в котором каждый элемент имеет ссылку на предыдущий элемент и на следующий.
Добавление и удаление элементов в середине LinkedList прооисходит с алгоритмической сложностью O(1), когда в ArrayList это делается за O(n). Поиск и удаление по индексу в ArrayList делается за O(1), а в LinkedList за O(n).
```

- Как устроена коллекция HashSet?
```
В основе HasSet лежит хеш-таблица. В обычных таблицах элементы идут подряд, без пропусков, а в хеш-таблице элементы могут лежать в разных ячейках, и распределяться они будут на основании хеш-кода. В HashSet изначально находится массив из 16 ячеек, и когда мы добавляем объект, вычисляется остаток деления хеш-кода этого объекта на число 16, и на основе этого остатка определяесятя место в массиве. Но в сами ячеки мы помещаем не наш объект, а объект класса Entry, который содрежит два поля - значение нашего объекта и сслыка на следующий объект Entry, который будет добавлен в случае, если нам поподется объект с таким же хеш-кодом. Это называестя коллизией. Ячеки в массивах работают как связный список, но для оптимизации также есть специально поле с плавающей точкой, в котором указан коэфицент 0.75. Это означает, что если массив будет заполнен на 75%, то массив увеличиватся в два раза.
```
